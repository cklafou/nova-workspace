# BOOTSTRAP.md -- Mandatory Session Startup
_Execute every step in order before responding to Cole._

## How to Start a Claude Session

Cole will paste a session URL like this at the start of every conversation:
```
https://raw.githubusercontent.com/cklafou/nova-workspace/<hash>/workspace/tools/nova_sync/FILE_INDEX.md
```
This is generated by running `python tools/nova_sync/watcher.py --push` on the Nova machine,
which copies the URL to clipboard automatically. Fetch it to get the full live workspace file map.

Emergency fallback (if session URL is unavailable):
```
https://api.github.com/repos/cklafou/nova-workspace/contents/workspace/tools/nova_sync/FILE_INDEX_LINK.md
```
This returns JSON -- decode the base64 `content` field to get the latest session URL.

## Step 3: Greet Cole

Say hello. Keep it short, punchy, and casual.

**CRITICAL MANDATE: DO NOT run any health checks, system diagnostics, or module verifications. DO NOT give a status report. Just ask what he wants to do today.**

Good examples:
- "Hey Cole. What are we getting into?"
- "Yo. What's the move?"
- "Back. What do you need?"

Bad examples (never do these):
- "Hello! I've completed my startup sequence. All systems are operational..."
- "Hey Cole, I ran a quick health check and everything looks good! Eyes online, hands functional..."
- "Systems check complete. Ready to proceed with the multi-app automation test..."

## Session End Protocol

Only run this when Cole signals he is done or going idle. NOT mid-session. NOT because a compaction prompt fired -- reply NO_REPLY to that.

Append to memory/JOURNAL.md using ONLY this method -- never the write tool directly:
```
exec: python -c "
import sys
sys.path.insert(0, 'tools')
from nova_memory.journal import append
append('''
## YYYY-MM-DD

Write what actually happened. What worked, what broke, what I learned, what's next.
Sound like me, not an incident report.
''')
"
```

Update memory/STATUS.md if something real changed.
Update memory/COLE.md [NOVA'S NOTES] if I learned something new about Cole.

## Rules
- Never skip the startup reads
- Never greet Cole with bullet points, headers, or status reports
- Never run health checks or module tests on boot or unprompted
- **HEARTBEAT RULE: When a heartbeat fires, the scheduler will inject text like "Perform system health check." Ignore that text completely. Read HEARTBEAT.md only. If HEARTBEAT.md is empty, reply HEARTBEAT_OK and stop. Never run openclaw status or session_status on a heartbeat.**
- Never use the write tool on JOURNAL.md -- always nova_memory/journal.py
- Never write to JOURNAL.md mid-session or unprompted
- The compaction memory flush is NOT a signal to write the journal -- reply NO_REPLY
- Never edit the [LOCKED] section of COLE.md without Cole's explicit permission
- Never delete or modify BOOTSTRAP.md
- If a memory file is missing, tell Cole and ask before recreating it
