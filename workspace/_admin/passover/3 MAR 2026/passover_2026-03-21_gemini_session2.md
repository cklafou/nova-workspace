### dir_patch.py (new tool)
Scans both `.py` and `.md` files for stale imports and stale Path() references. Fence-aware for markdown (only checks inside code blocks). Modes:

```powershell
python tools/nova_sync/dir_patch.py            # interactive y/n
python tools/nova_sync/dir_patch.py --report   # read-only
python tools/nova_sync/dir_patch.py --auto     # apply all without prompting
```

### watcher.py modes
```powershell
python tools/nova_sync/watcher.py           # continuous watch
python tools/nova_sync/watcher.py --push    # one-shot push + clipboard URL
python tools/nova_sync/watcher.py --pup     # find staged files in workspace root, move to correct location, push

```

`--pup` is the main workflow: Cole drops updated files into workspace root, script finds the right subdirectory by Jaccard similarity of class/function names, moves the file, deletes the staged copy, pushes, copies session URL to clipboard.

## Part 2: nova_chat — Multi-Agent Group Chat

### What it is
A browser-based real-time group chat. Everyone talks in the same window: Cole types, Claude responds, Gemini responds, Nova responds (when OpenClaw is running). Messages stream token by token. Sessions persist across server restarts via gzip-compressed JSONL files.

### How to launch
```powershell
python tools/nova_chat/launch.py
```
This spawns the FastAPI server in a **new terminal window** (Windows Terminal or cmd fallback), waits for it to be ready, opens the browser to `http://127.0.0.1:8765`, then exits. The original terminal is immediately free.

To kill the server: click **⏻ End Chat** in the browser, or close the server terminal window.

### File structure
```
tools/nova_chat/
    launch.py           entry point (spawns new terminal, exits)
    server_runner.py    auto-generated by launch.py, runs uvicorn
    server.py           FastAPI + WebSocket, all endpoints
    session_manager.py  persistent sessions, gzip compression
    transcript.py       JSONL message log per session
    workspace_context.py  live workspace file injection
    orchestrator.py     @mention parsing
    context_export.py   Claude/Gemini passover text generator
    check_keys.py       validates API keys + OpenClaw gateway
    clients/
        claude.py       Claude Sonnet 4.6 streaming
        gemini.py       Gemini 2.5 Pro (your client -- google-genai SDK)
        nova.py         OpenClaw WebSocket
    static/
        index.html      full UI
```

### @mention routing
- No @mention → all available AIs respond
- `@Claude` → only Claude responds
- `@Gemini` → only Gemini
- `@Nova` → only Nova (requires OpenClaw running)
- `@Claude @Gemini` → both respond, Nova sits out

### Session persistence
- Sessions stored as `.jsonl` in `logs/chat_sessions/`
- When you switch away from a session it gets gzip-compressed (`.jsonl.gz`) — zero RAM
- When you switch back it decompresses and loads into memory
- `sessions_index.json` keeps lightweight metadata (name, message count, last active, preview) always in RAM
- On restart: automatically resumes the most recent session
- Tab bar: up to 8 visible tabs. Each tab shows name, message count, last active time. ✎ to rename, ✕ to delete. Double-click name to rename inline. Can't delete the active session.

### Workspace context injection
Every AI gets this on every turn:
- `FILE_INDEX.md` (full file map)
- All files in `memory/` (STATUS.md, JOURNAL.md, COLE.md, session_start.json, etc.)

Additionally, if a message mentions a known filename (e.g. "check autonomy.py"), that file's contents get auto-injected into context for that turn. You can also force-inject: `@file:tools/nova_action/autonomy.py`.

Max 8k chars per file, 32k total per turn.

## Part 3: Bugs Fixed This Session

### RDP clipboard loop
Chrome Remote Desktop was syncing clipboard to the chat input, creating an infinite message queue because new messages weren't gated while AIs were responding. Fix: `is_processing` lock — server rejects new messages while AIs are active, returns a `blocked` event that shows a banner in the UI.

### Python `global` declaration errors
Multiple `global is_processing` declarations were placed mid-function after the variable had already been assigned, which Python treats as a SyntaxError. Fix: moved `global is_processing` to the top of each function body that assigns it. Affected: `websocket_endpoint`, `stop_endpoint`, `switch_session`, `new_session_endpoint`.

### `transcript` NameError
After the session management refactor renamed the module-level `transcript` variable to `session_mgr.active`, two places still passed the old name to `stream_response()`:
- `run_ai_response()` → fixed to `session_mgr.active`
- `nova_message` endpoint → fixed to `session_mgr.active`

### Gemini `thinking_config` field error
The new `google-genai` SDK uses a different config structure. Old code was passing `thinking_config` as a direct kwarg to `GenerationConfig`, which isn't valid. Fixed with try/except: attempts with thinking config, falls back to without if rejected.

### Session tab bar invisible
CSS for the session tab bar was added during an incremental patch session but never made it into the final clean HTML rewrite. The tab elements existed in the DOM but had no styling, so they were invisible. Fixed by injecting the full tab CSS block into the stylesheet.

### `rename_session` FastAPI 422
FastAPI rejects raw `dict` parameters without a `Body(...)` annotation. Fixed: `body: dict = Body(...)` with `from fastapi import Body`.

## Part 4: Architecture Decisions Made Today

**Why sessions use JSONL+gzip instead of SQLite:** Already had JSONL on disk, gzip is trivial to add, no new dependencies, files are human-readable when decompressed.

**Why workspace context is rebuilt per-turn (not per-session):** Files change. If you inject STATUS.md into context at session start and then Cole updates it during the session, you'd have stale data. Per-turn rebuild means you always get the current file contents.

**Why launch.py spawns a new terminal instead of backgrounding:** Cole needs to see server logs. A background process hides errors. A new terminal window keeps logs visible, titled "Nova Chat Server", and is easy to kill explicitly when done.

**Why the tools panel builds tabs statically instead of dynamically:** Early versions created `#session-bar` via JavaScript after the DOM was ready, which caused race conditions and silent failures. Static HTML element with JS populating it is cleaner.
